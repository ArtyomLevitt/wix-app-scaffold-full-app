# wix-app-scaffold-full-app

Cloud-runtime scaffold for **Via Agent** builds (Cursor SDK + Composer 2.5).

When the App Factory's "Via Agent" pill is used, the Cursor SDK clones this
repo into an isolated cloud VM and runs Composer 2.5 against the merchant's
spec. The agent reads the canonical Foundation skills bundled below at
`.cursor/skills/`, generates the app per spec, and opens a PR.

## What this scaffold is for

Full Wix app with dashboard page, site widget, optional editor settings panel, and backend events.

## Hands-off rules the agent will follow

The agent is configured to obey every rule in:

```
.cursor/skills/wix-app-foundation-system-blueprint/SKILL.md
.cursor/skills/wix-cli-orchestrator/SKILL.md
```

…plus the supabase-app-schema, ai-corrections, dashboard-page, and other
category-shared skills under `.cursor/skills/`.

## After the agent finishes

The Foundation pipeline:

1. Waits for `run.wait()` to resolve.
2. Clones the agent's PR head locally.
3. Runs the same smoke harness used for the phased pipeline
   (`lib/app-factory/smoke-harness.js`).
4. On smoke failure, calls `Agent.resume()` with synthesized fix instructions
   (capped at 1 retry, matching the existing auto-tweak loop).
5. Marks the build complete / failed and surfaces the PR URL + cost in the
   App Factory detail page.

## Do NOT edit this scaffold by hand

It's regenerated by:

```bash
node scripts/build-cursor-scaffold.js 
```

…inside the `purple-wix-connect` repo. Any manual edits here will be lost on
the next regeneration.
